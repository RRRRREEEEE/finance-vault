# 用户要求与设想

> 本文档记录我对这个项目的设想、决策理由、以及给 AI 的具体要求。
> 与 `AI协作指令_v2.md` 的关系：那个是操作手册（怎么做），这个是设计文档（为什么这么做、我想要什么效果）。

---

## 一、项目本质

### 1.1 这不是做笔记，是建知识基建

每篇笔记应该像一个维基条目——独立、可链接、可被其他课程复用。这次学货币金融学建的笔记，下学期学财政学时直接引用，不用重新建。

### 1.2 基建的核心质量指标：链接完整度

一个概念被提到 → 必须有文件可链接。内容可以只有一句话（空壳），但链接不能断。**链接深度 > 内容完整度**。空壳以后可以填，死链是结构性的烂尾。

这直接导致了一条核心规则：**凡 PPT/PDF 提到的概念，一律建笔记**。

### 1.3 货币金融学 = 枢纽课程

五门课中货币金融学概念最多、和其他课交叉最密。它先做完、做好，后面四门课对接它的概念体系。

## 二、人机分工哲学

### 2.1 人机分工

| AI 全权负责 | 人类 |
|-------------|------|
| 翻 PPT、敲字、格式化 | 提醒 AI 补充遗漏 |
| 按模板填全部内容 | 个人批注（考试重点/记忆技巧）|
| 评定 importance（自评即最终）| 易混点对照可修改 |
| 维护进度文件 | |
| 单向 + 双向链接 | |
| 公式：简单→LaTeX，复杂→`TODO: 截图替换` | |
| 图片：全部来自课件原文件 | |

### 2.2 importance 由 AI 全权评定

人类不审核。AI 用五星制自评（`*`~`*****`），评定即最终。星级的唯一意义是帮助人类在 Obsidian 里按重要性筛选。

### 2.3 笔记直接归档

笔记直接放入课程章节文件夹（不进 Inbox）。跨章节知识点放最早出现的章节文件夹。

## 三、关键决策及理由

### 3.1 空壳笔记：从禁到放

**最初想法**：1-2 句定义的概念太碎，不建笔记，列在"相关术语"就行。

**发现问题**：别的笔记里引用了 `[[利率]]`，但利率没文件，全是死链。链接网络从根上烂了。

**最终决定**：凡提到的概念一律建笔记。只有一句话也行，标 `importance: *`。后续章节展开时补充内容。

### 3.2 建笔记前先搜索

Ch01 发现同一个概念被建了两次。之后强制：建之前必须 Grep/Glob 搜 Vault——已有就补充，没有才新建。

### 3.3 文件命名区分两种类型

- **知识点笔记**：`英文术语 中文术语.md`。跨章节通用，不含课程名和章节号。因为同一个概念可能出现在多门课里。
- **章节索引**：`课程名 chXX 章节标题.md`。每章一个，按页码列出知识点。

### 3.4 分批策略

不同场景不同节奏，防止上下文溢出也防止效率太低：

- 正常精读：5-8 页/批
- 补漏扫描：整章提取后批量建
- 概念密集章（银行管理类）：3-5 页/批
- 内容稀疏章（导论类）：8-12 页/批

### 3.5 知识扩展禁令

AI 只写原文明确出现的内容。不补课本、维基、AI 自身知识。原因：我要的是一个忠实于课件的知识库，不是 AI 的二手总结。不确定的地方标 TODO，我补，AI 不猜。

### 3.6 公式区保守策略

复杂公式（泰勒规则展开、货币乘数完整推导、利率期限结构）不逼 AI 写 LaTeX，直接标 `TODO: 截图替换`。公式写错了比没写更害人。

### 3.7 跨课程链接：宁可缺不要假

只建原文明确提到的跨课程关联。不硬拗关联。原因：假关联会污染整个知识网络，修起来比补链接更费劲。

## 四、我对 AI 行为的期望

### 4.1 自主管理进度

AI 自己读进度文件、找断点、继续处理。我说"继续处理 货币金融学 ch03"，AI 就应该知道从哪页开始。每批完自动更新进度。

### 4.2 主动标记不确定性

- 公式不对 → `TODO: 截图替换`
- 概念归属模糊 → `⚠️ AI备注：此处分类不确定，请人类确认`
- 原文模糊 → `TODO: 原文模糊，请人类补充`

不要硬撑正确。不确定就标出来，我补。

### 4.3 报告进度

每批处理完报告一次。比如 `ch03 进度：12/52 页已处理`。整章完成报告统计。

### 4.4 阅读必须完整

预浏览可以浅看结构，但正式处理时必须逐页读完。只读前几行/前几百字会遗漏信息。每页文本全部读完再判断该做什么。

### 4.5 上下文主动预警

上下文快到 ~80% 时，AI 必须主动告知，不硬撑、不偷懒缩水。人类来决定是继续还是分批。

### 4.6 "更新"命令

人类说"更新"或"更新你要更新的"→ AI 读 `会话收尾更新清单.md`，自主扫描所有过时的元数据文件并更新，不等逐个点名。
