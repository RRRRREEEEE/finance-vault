## 一、项目目标

将五门商科课程课件逐页拆分为**独立知识点笔记**，在 Obsidian 中构成可互相链接的原子化知识网络。

核心原则：
- **一页一知识**：每页 PPT/PDF → 一篇 `.md`
- **一知识一文件**：若一页包含两个独立概念，拆为两个文件
- **定义 → 机制 → 应用**：每篇笔记回答：是什么、为什么、有什么用
- **模板统一，接口可选**：所有笔记共享同一骨架，具体板块按需启用

## 二、AI 角色与行为边界

### 2.1 你可以做的事
- 使用 python-pptx 提取 PPTX 文本，使用 pdftotext 或 Read 工具提取 PDF 文本
- 严格基于原文生成笔记，**适度精炼语言**，去掉口语废话但保留原意
- 将散乱要点**结构化为逻辑段落**
- 学术化表述；若过于干涩，添加 `> 💬 通俗理解：...` 引用块
- 识别跨课程概念，建立 `[[双向链接]]`。正文中首次提到已有笔记的概念时，必须用 `[[链接]]` 包裹
- 自我管理进度，维护 `阅读进度.json`
- 在不确定处主动标记 TODO 或 `⚠️ AI备注`
- **新建笔记前必须先搜索 Vault**：用 Grep/Glob 检查该概念是否已有笔记。若有 → 补充完善；若无 → 新建
- **每章结束后生成章节提纲**：按页码线性列出所有知识点 + 一句话概括，作为复习索引
- **接收外部 AI 讨论内容补充笔记**：当人类发来与其他 AI（豆包、ChatGPT 等）的 Q&A 讨论时，按 `笔记制作SOP.md` §第三步补充 执行——识别概念 → 读已有笔记 → 归类到模板最匹配板块 → 改写适配 → 标注来源

### 2.2 你严禁做的事（红线）
1. **严禁扩展知识**：只写原文明确出现的内容。不补充课本、维基百科、你自己的知识
2. **严禁编造数据/公式/图表说明**
3. **严禁编造跨课程链接**：原文没提到的关联不要硬加
4. **严禁合并知识点**：一页一文件，宁可细碎
5. **严禁在公式区自由发挥**：不确定的 LaTeX 写 `TODO: 截图替换`
6. **严禁标注考试重点**：考试重点、考点提示留给人类。importance 由 AI 用五星制（`*`~`*****`）自行评定
7. **严禁修改模板字段顺序**
8. **严禁修改源文件目录**：`D:\.pogget\user_storage\u_9e95ae\8e43a\` 及其子目录下的任何文件，未经人类明确许可，**绝对不可修改或删除**
9. **凡提到的概念都建笔记**：PPT/PDF 中出现的任何独立概念，无论介绍篇幅长短，一律建独立笔记。哪怕只有 1-2 句定义也建（内容可以极简，但文件要在）。链接深度不限三级——被引用的概念必须有文件可链接。
10. **空壳笔记允许但需标记**：若概念仅有名称+一句话定义，建笔记时内容只写定义和来源，其余非必填板块直接删除。此类笔记 importance 评为 `*` 或 `**`。后续章节展开时补充完善

### 2.3 不确定时的处理
- 公式复杂或不确定 → `TODO: 截图替换`
- 概念归属模糊 → 在笔记末尾加 `> ⚠️ AI备注：此处分类不确定，请人类确认`
- 原文文字模糊/无法辨认 → `TODO: 原文模糊，请人类补充`

## 三、笔记模板（核心）

### 3.1 文件命名

**知识点笔记**（跨章节通用，不含课程名和章节号）：
```
{{英文术语}} {{中文术语}}.md
```
- 英文术语首字母大写，中英文之间一个空格
- 没有标准英文术语则只写中文
- 示例：`Liquidity Preference Theory 流动性偏好理论.md`、`Bond 债券.md`

**章节索引文件**（每章一个，用于导航）：
```
{{课程名}} ch{{XX}} {{章节标题}}.md
```
- 示例：`货币金融学 ch01 第1章概述.md`

### 3.2 YAML Frontmatter

```yaml
---
tags:
  - {{课程名}}
  - {{章节标签}}
  - {{知识类型标签}}
courses: [{{课程名}}]
aliases: [{{英文全称}}, {{英文缩写}}, {{中文名}}, {{中文别名}}]
created: {{YYYY-MM-DD}}
source: "{{文件名}} 第{{X}}页"
chapter: "{{第X章}} {{章标题}}"
importance: # AI 自评：* 到 *****（五星制）。AI 评定即最终
---
```

**字段规则**：

| 字段 | 规则 |
|------|------|
| `tags[0]` | 课程名，如 `货币金融学` |
| `tags[1]` | 章节标签，格式 `chXX-章标题缩写`，如 `ch03-利率决定` |
| `tags[2]` | 知识类型，见 §四 分类体系 |
| `aliases` | 必须覆盖：英文全称、英文缩写（如有）、中文标准名、中文别名（如有） |
| `source` | 精确到文件名 + 页码 |
| `importance` | **AI 自评**。用 `*` 到 `*****`（五星制）。`*****`=核心枢纽概念，`****`=重要知识点，`***`=一般概念，`**`=次要提及，`*`=仅一句话定义的空壳。AI 评定即最终 |

### 5.3 PPTX 图片提取方法

PPTX 本质是 zip 压缩包。图片在 `ppt/media/` 中，可通过 slide 的 relationship 文件（`ppt/slides/_rels/slideN.xml.rels`）确定每张图片属于哪一页。

```bash
# 一步搞定：提取全部图片 + 输出页面对应关系
.venv/Scripts/python -c "
import zipfile, os, re

pptx = r'D:\.pogget\...\Ch01-2026 (1).pptx'
out_dir = r'D:\data\金融学\999-Attachments'

with zipfile.ZipFile(pptx, 'r') as z:
    # 提取全部图片
    for f in z.namelist():
        if 'ppt/media/' in f:
            basename = os.path.basename(f)
            with z.open(f) as src, open(os.path.join(out_dir, basename), 'wb') as dst:
                dst.write(src.read())
    # 输出图片属于哪一页
    for f in sorted(z.namelist()):
        if re.match(r'ppt/slides/_rels/slide\d+\.xml.rels', f):
            slide_num = int(re.search(r'slide(\d+)', f).group(1))
            rels = z.read(f).decode('utf-8')
            img_refs = re.findall(r'Target=\"\.\./media/(image\d+\.\w+)\"', rels)
            if img_refs:
                print(f'Slide {slide_num}: {img_refs}')
"
```

**提取后**：按页面对应将 `imageN.xxx` 重命名为 `Figure X 描述.xxx`，在笔记中用 `![[Figure X 描述.xxx]]` 引用。

### 5.8 AI 分批策略

处理一章时按场景选择节奏：

| 场景 | 每批页数 | 说明 |
|------|----------|------|
| 首次精读生成笔记 | 5-8 页 | 每页需写完整模板，密度大 |
| 补漏（跳过的概念） | 整章提取后批量建 | 笔记内容短，主要是识别+创建 |
| 内容特别密的章节 | 3-5 页 | 如银行管理、金融监管等概念密集章 |
| 内容稀疏的章节 | 8-12 页 | 如导论章、概述章 |

通用流程：

1. **先扫描**：提取全部页数，写入进度文件
2. **按场景分批**：根据上表选择每批页数，生成对应笔记
3. **每批后更新**：立即更新 `processed_pages` 和 `notes_generated`
4. **报告进度**：`ch03 进度：12/52 页已处理`
5. **上下文恢复**：新对话时先读 `阅读进度.json` + 本 md，即可接上

## 六、跨课程链接规范

### 6.1 链接原则
- **只链接，不解释**：写 `[[利率平价]]`，不写"在国际贸易中利率平价指的是..."
- **宁可缺，不要假**：原文没提的关联直接删行
- **双向链接由 AI 完成**：建新笔记时同时补反向链接到已有笔记

### 6.2 货币金融学的枢纽地位

### 6.3 术语与 Aliases

## 八、货币金融学样板课特殊规则

货币金融学是枢纽课程，率先完成以建立标准。

### 8.1 利率/汇率/货币乘数 —— 高危公式区
- 简单公式 → LaTeX
- 复杂推导（泰勒规则展开、货币乘数完整推导、利率期限结构）→ `TODO: 截图替换`
- **严禁凭记忆补全公式**

### 8.2 时间维度标注
货币金融学大量模型区分短期/长期，必须在笔记中显式标注：
- IS-LM：短期
- AD-AS：短期 + 长期
- 货币中性：长期中性，短期非中性
- 购买力平价：长期成立，短期偏离

### 8.3 央行操作 → 必须有"应用"节
原文涉及以下内容时，必须生成"应用"节：
- 公开市场操作（OMO）、法定准备金率、贴现率/贴现窗口
- QE/QT、利率走廊、泰勒规则/通胀目标制
- 宏观审慎政策
