## 一、跨课程同一概念的处理

### 问题

同一个名词，不同课程讲法完全不同。例如：

| 概念 | 财政学 | 中级微观经济学 |
|------|--------|--------------|
| 边际成本 MSC | 画个图，MSC=MSB → 帕累托最优 | 完整推导：TC求导、短期长期成本曲线、计算题 |
| 垄断 Monopoly | 提一嘴"造成无谓损失" | 垄断定价模型、价格歧视、勒纳指数 |
| 外部性 Externality | 提一嘴"市场失灵原因" | 科斯定理、庇古税、排污权交易 |

### 方案：一笔记 + 课程分区

**一条概念只建一个文件**，内部按课程分节。不同课程的内容在同一文件里从上往下摞。

**模板**：

```markdown
---
tags:
  - 枢纽概念        ← 跨课程概念专属标签
  - 经济学基础
courses: [财政学, 中级微观经济学, 货币金融学]
aliases: [Marginal Cost, MC, 边际成本]
importance: *****
---
```

### 规则

1. **哪个课程先提到，就由哪个课程建文件**。后续课程往里加节。
2. **每节课的内容加 `## 🏫 XXX视角`** 小节，标注涉及章节。
3. **通用定义放最上面**。课程特有内容放各自的小节。
4. **加 `枢纽概念` 标签**。方便在 Obsidian 里搜 `tag:枢纽概念` 看到所有跨课程概念。
5. **aliases 必须覆盖所有可能的叫法**。财政学叫 MSC、微观叫 MC、中文叫边际成本——全部写进 aliases。

### 什么概念值得单开枢纽笔记

判断标准：**出现于 ≥2 门课，且有实质内容≥1 门课**。

| 条件 | 处理 |
|------|------|
| 多门课出现 + 至少一门有深入讲解 | 建枢纽笔记，按课程分节 |
| 只在一门课出现 + 有深入讲解 | 放该课程文件夹，普通笔记 |
| 多门课出现 + 都是提一嘴 | 壳笔记，标 `**`，待后续补充 |
| 只在一门课出现 + 提一嘴 | 壳笔记，放该课程文件夹 |

## 二、重复笔记的预防与清理

### 2.1 事前预防（AI 必须执行）

**建笔记前先搜索**（已有规则，重申）：

```bash
# AI 新建笔记前必须执行
Grep "笔记名"  # 全文搜
Glob "**/笔记名*"  # 文件名搜
```

搜索范围：整个 Vault，不限于当前课程文件夹。

### 2.2 定期清理：每月一次

建议用 Dataview 插件在 Obsidian 里建一个仪表盘笔记。以下代码直接可用：

#### 检测同名文件

````markdown
```dataviewjs
const files = app.vault.getMarkdownFiles()
const nameMap = {}
for (const f of files) {
  const name = f.name
  if (!nameMap[name]) nameMap[name] = []
  nameMap[name].push(f.path)
}

const dupes = Object.entries(nameMap).filter(([k,v]) => v.length > 1)
if (dupes.length === 0) {
  dv.paragraph("✅ 无重复文件名")
} else {
  dv.header(2, `⚠️ ${dupes.length} 组重复文件名`)
  for (const [name, paths] of dupes) {
    dv.paragraph(`**${name}**: ${paths.map(p => `[[${p}]]`).join(' / ')}`)
  }
}
```
````

#### 检测 aliases 冲突

````markdown
```dataviewjs
const files = app.vault.getMarkdownFiles()
const aliasMap = {}

for (const f of files) {
  const fm = app.metadataCache.getFileCache(f)?.frontmatter
  const aliases = fm?.aliases || []
  for (const a of aliases) {
    if (!aliasMap[a]) aliasMap[a] = []
    aliasMap[a].push(f.path)
  }
}

const conflicts = Object.entries(aliasMap).filter(([k,v]) => v.length > 1)
if (conflicts.length === 0) {
  dv.paragraph("✅ 无 aliases 冲突")
} else {
  dv.header(2, `⚠️ ${conflicts.length} 个 aliases 被多个笔记使用`)
  for (const [alias, paths] of conflicts) {
    dv.paragraph(`**${alias}**: ${paths.map(p => `[[${p}]]`).join(' / ')}`)
  }
}
```
````

### 2.3 清理流程

发现重复后：

1. **判断是真重复还是凑巧同名**
   - 同名同概念 → 合并，旧文件重定向到新文件
   - 同名不同概念 → 文件名加区分后缀
2. **合并时保留更完整的版本**，用 `[[旧文件名]]` 做重定向
3. **aliases 合并**：把所有叫法汇总到保留的那个文件里

### 2.4 建议的插件

| 插件 | 用途 |
|------|------|
| **Dataview** | 上面两个查重脚本的基础 |
| **Alias Management** | 自动检测 aliases 冲突，实时标记 |
| **Note Refactor** | 合并/拆分笔记时用 |
| **Tag Wrangler** | 批量重命名标签 |

## 三、定期维护计划

### 建议节奏

| 频率 | 执行内容 | 执行方式 |
|------|----------|----------|
| **每次建笔记前** | 搜索 Vault 避免重复 | AI 执行 |
| **每章完成后** | 检查本章 aliases 是否完整 | AI 提醒 |
| **每门课完成后** | 跑 Dataview 重复检测 | 人类在 Obsidian 里打开仪表盘看一眼 |
| **每 2-3 门课后** | 审查枢纽概念标签、合并跨课程重复 | 人类 + AI 协作 |
| **项目全完成后** | 全面去重、aliases 统一、标签整理 | 人类主导 |

### 枢纽概念仪表盘

建一个 `枢纽概念索引.md` 放在 `900-跨课程枢纽/`：

````markdown
```dataview
TABLE courses, importance, length(file.outlinks) as "链接数"
FROM #枢纽概念
SORT importance DESC
```
````

这样你在 Obsidian 里打开就能看到所有跨课程概念的列表，按重要性排序。
