## 一、笔记存放位置错误

### 问题
ch04 创建的 15 篇笔记全部放在了 Vault 根目录（`D:\data\金融学\`），而不是课程文件夹内。

### 正确做法
- 笔记应放在对应课程的章节文件夹中：`{课程编号}-{课程名}/ch{XX}-{章节标题}/`
- 例如：`200-财政学/ch04-外部性与公共产品/`
- **只有**跨课程枢纽概念可以放在课程文件夹之外（但首次出现章节的文件夹优先）

### 根因
AI协作指令_v2.md 虽然写了 Vault 结构，但没有在"建笔记"步骤中显式要求指定目标路径。AI 默认写到了 vault 根目录。

### 标准化
**新建笔记前必须确定目标路径**：
1. 单课程概念 → `{课程文件夹}/ch{XX}-{章节}/`
2. 跨课程概念（枢纽）→ 首次出现课程的章节文件夹；不确定时放入当前课程
3. 壳笔记 → 与内容笔记同路径
4. 章节索引 → `{课程文件夹}/ch{XX}-{章节}/财政学 ch{XX} {章节标题}.md`

## 二、PPTX 图片处理不完整

### 问题
ch04 笔记中 6 处图表使用了 `TODO: 截图替换` 占位符，没有插入实际图片。

### 正确做法（按优先级）
| 图片来源 | 工具 | 方法 |
|----------|------|------|
| PPTX 嵌入图片（image/png/jpeg）| zipfile | 从 `ppt/media/` 提取，按页面对应关系重命名为**描述性名称** |
| PPTX 嵌入矢量图（EMF/WMF）| PowerShell .NET | 用 `System.Drawing.Image` 读取 → 绘制到 Bitmap → 保存为 PNG |
| PPTX 绘制图形（AutoShape/Line）| **matplotlib 重新绘制** | 根据文本描述（轴标签、曲线名、交点位置）用 matplotlib 生成等效图表 |
| PDF 嵌入图 | PyMuPDF | `page.get_images()` → `extract_image()` |
| PDF 矢量图/页面渲染 | PyMuPDF | `page.get_pixmap(dpi=200)` 渲染为 PNG |

### 图片命名规范
```
{课程名}Ch{XX} {描述性中文名称}.png
```
- 示例：`财政学Ch04 负外部性的经济影响.png`
- **严禁**使用 hash 命名（如 `财政学ch04_773fdc18.png`）
- 章节索引中的大写 Ch + 两位数字

### 根因
1. AI协作指令_v2.md §5.3 只覆盖了 zip 提取嵌入图片，未覆盖 EMF 和绘制图形的处理
2. §5.5"图表生成"写了用 matplotlib 但没有和"图形提取不出来的情况"建立关联
3. 图片命名规范隐含在 §5.3 末尾"重命名为 Figure X 描述.xxx"，但没有大写 Ch 编号的强制格式

### 标准化
- 提取 PPTX 图片后**立即重命名**为描述性名称，不要保留 hash 名
- 绘制图形（AutoShape）→ 🚫 **严禁用 matplotlib 自己画** → 用 PowerPoint COM 导出该页为 PNG
- PowerPoint COM 导出方法：`win32com.client.Dispatch('PowerPoint.Application')` → `slide.Export(path, 'PNG', 1920, 1080)`
- EMF 嵌入图 → 用 PowerShell `System.Drawing.Image` 转换（比全页导出更精确）
- 图片永远用课件原图

## 二点五、用 matplotlib 自绘图表（🚫 严重错误）

### 问题
ch04 处理时对 PPTX 中 AutoShape 绘制的图表，用 matplotlib 自行重新绘制了 4 张图表。

### 为什么错误
用户明确要求用课件里的图。AI 自己画的图与课件原图不一致，且丢失了课件的教学设计和排版信息。

### 正确做法
用 **PowerPoint COM** 导出整页为高清 PNG（见上节标准化）。
- 🚫 **绝对禁止**用 matplotlib 重新绘制课件图表
- PPTX 无嵌入图 → PowerPoint COM 导出整页
- PDF 无嵌入图 → PyMuPDF `page.get_pixmap()` 渲染整页
- 图片永远是课件原图

## 三、章节文件夹命名不一致

### 问题
ch04 章节索引放在了 `200-财政学/` 而不是 `200-财政学/ch04-外部性与公共产品/` 里面。

### 正确格式
```
200-财政学/
├── ch01-财政变迁/
│   ├── （知识点笔记）.md
│   └── 财政学 ch01 财政变迁.md    ← 章节索引在文件夹内
├── ch03-福利与财政/
│   ├── （知识点笔记）.md
│   └── 财政学 ch03 福利与财政.md
└── ch04-外部性与公共产品/
    ├── （知识点笔记）.md
    └── 财政学 ch04 外部性与公共产品.md
```

### 标准化
- 文件夹名：`ch{XX}-{章节简称}`（无空格）
- 章节索引：`{课程名} ch{XX} {章节标题}.md`（放文件夹内）
- Ch 编号大写，两位数字（ch04 不是 ch4）

### 问题
有可用图片但笔记中写的是 `TODO: 截图替换`，没有实际 `![[图片名]]` 引用。

### 根因

### 标准化
**图片处理完成后再写笔记**（或写完立刻回填）：
1. 先提取/生成所有图片 → 命名 → 放入 999-Attachments
2. 再写笔记，在"图形"节直接用 `![[图片名]]` 引用
3. 写完检查：Grep 搜索 `TODO: 截图替换`，应返回 0 结果
