# 课程毕业标准

> **用途**：定义笔记/章节/课程"真正做完"的标准。防止 AI 偷懒导致质量参差不齐。
> **维护规则**：每门课完成时按此标准自检，不满足的标 `⚠️ 待补强`。
> **制定日期**：2026-06-20

---

## 一、单篇笔记最低标准（底线）

每篇笔记**至少**满足以下全部条件。不满足 → `⚠️ 待补强`，计入 `check_vault.py` 警告。

| # | 检查项 | 验证方式 |
|---|--------|----------|
| 1 | 📌 定义节有内容 | 至少 1 句完整定义，不能为空 |
| 2 | 📄 来源原文节有内容 | 至少 1 句原文摘录 |
| 3 | 🔗 至少 1 个 wiki 链接 | 正文或相关术语中至少 1 个 `[[]]` |
| 4 | 🏷️ frontmatter 7 字段齐全 | tags/courses/aliases/created/source/chapter/importance |
| 5 | ⭐ importance 有引号 | `"*"` ~ `"*****"` |
| 6 | 🚫 无 TODO 残留 | `TODO: 截图替换` / `TODO: 原文模糊` = 0 |
| 7 | 🖼️ 提到的图有实际文件 | `![[]]` 引用指向 `999-Attachments/` 中存在的文件 |

## 二、壳笔记标准（`*` 或 `**`）

壳笔记（仅 1-2 句定义的概念）底线更宽松：

| # | 检查项 |
|---|--------|
| 1 | 📌 定义节有那 1-2 句 |
| 2 | 📄 来源原文节标明出处 |
| 3 | 🏷️ frontmatter 字段齐全 |
| 4 | ⭐ importance 有引号（`"*"` 或 `"**"`） |

> 壳笔记不需要链接（内容太少，链了也是空的），不强制有机制/应用等可选节。

## 三、章节完成标准

一章标记 `completed` 前，必须满足：

| # | 检查项 | 说明 |
|---|--------|------|
| 1 | 每页 PPT/PDF 都已处理 | `processed_pages` 全覆盖 |
| 2 | 每篇笔记满足最低标准 | 跑 `check_vault.py --course XXX` 零 error |
| 3 | 章节索引已生成 | `{课程} ch{XX} {标题}.md` 含按页码排序的知识点列表 |
| 4 | 无孤概念 | PPT 出现过的概念全有文件（哪怕壳） |
| 5 | 阅读进度 JSON 已更新 | status = `completed` |

## 四、课程完成标准

一门课标记 `completed` 前：

| # | 检查项 |
|---|--------|
| 1 | 全部章节满足章节完成标准 |
| 2 | 全部源文件已登记到 `源文件清单.md` |
| 3 | 全部概念已登记到 `术语登记表.md` |
| 4 | `check_vault.py` 全库扫描零 error（warning 可接受） |
| 5 | 课程概述笔记（如 `货币金融学总索引.md`）已生成 |
| 6 | CLAUDE.md 进度表已更新 |
| 7 | Git tag 已打（如 `v1.0-货币金融学`） |

## 五、质量等级

| 等级 | 条件 | 标识 |
|------|------|------|
| 🥇 优秀 | 定义 + 机制 + 应用 + 图形 + 3+链接 + 跨课程链接 | — |
| 🥈 合格 | 满足最低标准，有定义+来源+至少 1 链接 | — |
| 🥉 待补强 | 不满足最低标准 | `⚠️ 待补强` 标在笔记末尾 |
| 壳 | 仅定义，满足壳标准 | importance `*` 或 `**` |
