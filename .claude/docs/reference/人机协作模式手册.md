## 一、图片协作

### 1.1 人类在 Obsidian 中直接粘贴图片

**场景**：人类在 Obsidian 中 Ctrl+V 粘贴截图 → Obsidian 自动生成 `Pasted image YYYYMMDDHHMMSS.png` → 放在 Vault 根目录。

**触发词**：人类说"这个是 ![[Pasted image ...]] 我自主插入的图片，你重命名"

**AI 执行流程**：
1. 搜索 Vault 根目录找到 `Pasted image *.png`
2. 搜索哪个 `.md` 文件引用了它 → 确定图片上下文
3. 读目标笔记，理解图片内容和位置
4. 给图片起描述性名称：`{课程名}Ch{XX} {描述}.png`
5. 复制到 `999-Attachments/`（避免因 Obsidian 锁定无法移动）
6. 更新笔记中的引用（`![[Pasted image ...]]` → `![[新名称.png]]`）
7. 补写 `> 图形说明：...` 描述
8. 提醒人类删除 Vault 根目录的原 `Pasted image` 文件

**红线**：
- ❌ 不删除或移动 `Pasted image` 原文件（Obsidian 会锁定）
- ❌ 不乱猜图片内容——通过上下文和尺寸判断

**示例**：
```
人类粘贴 → ![[Pasted image 20260619141433.png]]（1446x419，在 Portfolio Choice Theory 中四大决定因素表下方）
AI 命名 → 货币金融学Ch05 资产需求四大决定因素.png
AI 写入 → ![[货币金融学Ch05 资产需求四大决定因素.png]]
         > 图形说明：资产组合选择理论的四大决定因素...
```

### 1.2 人类说"某笔记应该有图，你去找"

**场景**：人类指出笔记缺乏图像难以理解（如"这种静态分析没有图不好理解"）。

**AI 执行流程**：
1. 读笔记确定缺失什么图
2. 在源文件中定位对应页面 → 优先用第二补充笔记（`货币、银行和金融市场.pdf`，人类筛选过的更清晰）
3. 对可编辑 PDF：提取嵌入的原图（`page.get_images()`），不要整页渲染
4. 对矢量绘制：用 drawings 边界框 + `page.get_pixmap(clip=...)` 精确裁切
5. 命名：`{课程名}Ch{XX} {笔记名} {描述}.png`
6. 插入笔记 📊 图形节，补 `> 图形说明`
7. 清理临时预览文件

**常见错误**：
- ❌ 从 PPT 导出（人类倾向：第二补充 > PPT）
- ❌ 整页渲染 PyMuPDF `page.get_pixmap()` 不裁切（人类要的是独立图）
- ❌ 图片命名不统一

### 1.3 图片命名规范

| 场景 | 格式 | 示例 |
|------|------|------|
| 课件原图 | `{课程名}Ch{XX} {描述}.png` | `货币金融学Ch05 货币市场均衡.png` |
| 第二补充 | `{课程名}Ch{XX} {描述}.png` | `货币金融学Ch05 利率时间路径.png` |
| 标注来源 | 在图形说明末尾加 | `来源：第二补充笔记 第31页。` |

## 二、内容补充协作

### 2.1 人类发来一段知识 → AI 补充到笔记

**场景**：人类从学习资料里摘了一段文字发来，指定要补充到某个笔记。

**触发模式**：
> "这个补充在XXX这个笔记上。另外，以带有XXX命名的笔记之间的链接没有打好。"

**AI 执行流程**：
1. 读目标笔记 → 找到最匹配的模板板块（定义/机制/应用）
2. 用 Edit 精确插入（保留下方已有内容）
3. 若引入新概念 → Grep/Glob 搜 Vault → 已有则链接，无则建壳笔记
4. 人类提到的关联笔记 → 逐篇检查双向链接完整性 → 补回链
5. 补完后 Grep 验证所有 `[[XXX]]` 链接目标存在

**示例**：人类发来"Risk-averse/risk-seeker/risk-neutral"内容 → AI 补充到 `Risk 风险.md` 的 🧠 机制节 → 建 3 篇壳笔记 → 修复 11 篇风险命名笔记之间的双向链接。

### 2.2 人类说"这里没看懂"

**场景**：人类指出笔记中某节内容不清楚。

**AI 执行流程**：
1. 读笔记定位问题节
2. 回查学习笔记原文补全信息
3. 重写该节：加定义、加机制推导、加链接
4. 更新笔记的 source / 📖 相关术语 对应更新
5. 若涉及图形 → 执行 1.2 流程找图

**示例**：人类说"我没看懂利率风险结构里的购买力风险" → AI 将两句话引用块扩展为完整小节（定义+机制+费雪方程链接+考研标注），并更新定义从"三个因素"到"四个因素"。

## 三、链接与去重协作

### 3.1 人类指出链接缺失 → AI 全量修复

**场景**：人类发现笔记中提到的概念没有 wiki 链接。

**AI 执行流程**：
1. Glob 搜索 Vault 确认哪些概念已有笔记、哪些缺失
2. 对已有者 → Edit 补 `[[链接]]`
3. 对缺失者 → 建壳笔记（importance `*` 或 `**`）
4. 同步补回链：被链接笔记的 📖 相关术语 也要链回来
5. Grep 验证全链入度/出度

**示例**：Moral Hazard 的 🏛️ 应用节 8 个概念全无链接 → 3 个缺失建壳 → 8 个全部打通 → 补回链到 5 个被引笔记。

### 1.6 AI 图文错配 → 人类逐张纠正

**场景**：AI 从第二补充提取多张图，但部分图放错了位置（如把均衡概览图当成收入效应图，把价格水平效应图当成货币供给图）。

**根因**：第二补充文本中文编码损坏，AI 无法准确匹配"哪段文字描述哪张图"。

**AI 执行流程**：
1. 人类指出"XX的图放错了" → 立刻停止辩解，重新扫描对应页面文字与图片的位置关系
2. 按 `文字在图片附近` 原则重新匹配（如 p30 文字"价格水平效应"紧邻 img0 → 该图是价格水平效应）
3. 修正后汇报每张图的来源（页码+img序号），让人类可验证

**关键认知**：当无法读懂中文文本时，用图片与文字块的**物理位置**（y 坐标）来判断图文对应关系，而不是靠猜。

**示例**：p29 img1(844×642) 被 AI 误标为收入效应 → 人类纠正：那是均衡概览图 → 人类自己粘贴正确的收入效应图 → AI 重命名归档。同时 p30 img0/img1 搞反 → 人类纠正后 AI 交换。

### 3.2 人类指出重复笔记 → AI 合并

**场景**：同一概念存在两个 `.md` 文件。

**AI 执行流程**：
1. 对比两篇笔记：来源、importance、内容完整度
2. 保留更完整的版本
3. 将精简版的独特内容合并进去（标 📎）
4. 删除精简版文件 → 消除 Obsidian 链接歧义
5. 更新保留版 source 字段反映合并来源

**示例**：`Moral Hazard 道德风险.md` ch02 vs ch08 → ch08 更完整（*****），合并 ch02 保险市场视角 → 删除 ch02。

## 四、进度与断点协作

### 4.1 人类说"继续" → AI 自主接续

**流程**：
1. 读 `阅读进度.json` 找断点
2. 读 `AI协作指令_v2.md` 恢复模板
3. 从断点页继续提取/建笔记
4. 每批完立即更新进度 JSON

### 4.2 人类说"更新你要更新的" → AI 跑收尾清单

**流程**：读 `会话收尾更新清单.md` → 扫描所有过时元数据 → 逐项更新。

## 五、工具优先级

| 操作 | 首选工具 | 原因 |
|------|----------|------|
| 图片源 | 第二补充笔记 PDF | 人类筛选过，比 PPT 更清晰 |
| PDF 图片 | `page.get_images()` 提取嵌入原图 | 保真，不降分辨率 |

### 1.4 人类替换 AI 放的图

**场景**：AI 从第二补充提取了图放入笔记，人类觉得不够好，自己从第二补充截了更合适的图粘贴进来。

**AI 执行流程**：
1. 读笔记找到 `![[Pasted image *.png]]` 引用
2. 理解人类替换的意图——人类认为原图不合适，用新图覆盖
3. 同名覆盖 `999-Attachments/` 中的旧图（或重命名后替换引用）
4. 补写/更新 `> 图形说明`
5. 删除 Vault 根目录的原 `Pasted image`

**关键认知**：人类替换 AI 的图 ≠ AI 做错了。人类对第二补充更熟悉，知道哪张图最能表达。AI 应快速配合重命名归档，不辩解。

### 1.5 人类指出知识遗漏 → AI 整合建枢纽笔记

**场景**：人类发现第二补充的某块知识分散在多个笔记里或完全缺失，要求 AI 整合成一篇独立笔记。

**AI 执行流程**：
1. 搜索第二补充定位原始讨论（Grep 关键词 → 确定页码范围）
2. 提取该范围内的所有嵌入图（直接 `get_images()` 提取，不裁切）
3. 识别当前哪些笔记覆盖了此知识的碎片 → 写新笔记时链接而非搬运
4. 按模板创建枢纽笔记，整合全因素速查表 + 图 + 应用案例
5. 更新章节概述索引
6. 更新 `人机协作模式手册.md` 记录此模式

**示例**：人类说"第二笔记中详细讨论了均衡利率的变动，这值得在ch05单开一张笔记" → AI 搜索 p24-27 → 建 `Changes in Equilibrium Interest Rates 均衡利率的变动`（`*****`）→ 整合 $B_d$ 四因素/$B_s$ 三因素/费雪效应双曲线 + 两个应用。

**关键认知**：人类比 AI 更清楚知识的完整结构。当人类说"你遗漏了"→ 不要辩解"已经分散在别的笔记了"，直接整合建新笔记。
