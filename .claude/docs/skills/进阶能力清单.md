# 进阶能力清单（Skills）

> **用途**：记录 AI 可以做的锦上添花的能力。这些**不是每次必做**，而是触发式执行——人类说了才做，场景触发了才调。
> **与 SOP 的区别**：SOP 是必须做的生产线（建笔记），能力是做了更好的加分项（让知识库更好用）。
> **加载方式**：本文件按需手动读取，不自动加载。

## 一、知识串讲

> ~~**触发**：人类说"做串讲" / "写串讲" / "生成串讲"~~
> **2026-06-26 更新**：串讲已移入 SOP 第四步，整章完成后**自动生成**，不再需要人类触发。
> **性质**：整章完成后的必做增强

### 1.1 串讲是什么

一章一篇的**讲稿式复习文件**。把该章所有原子笔记重组为连贯叙事，用口语化语气把概念串成故事。读者看完串讲应该能回答"这一章到底讲了什么、为什么是这个逻辑"。

**与笔记的区别**：笔记是字典（查概念），串讲是课堂（学逻辑）。

### 1.2 触发时机

- 整章笔记全部完成（status = `completed`）
- 章节索引已生成
- 检查清单已通过（零 error）
- **人类明确说要串讲**

### 1.3 文件命名与路径

```
{课程编号}-{课程名}/ch{XX}-{章节简称}/Ch{XX} 知识串讲 {核心问题}.md
```

核心问题用一句话概括整章要回答的问题。示例：`Ch08 知识串讲 信息不对称如何塑造金融体系.md`

### 1.4 串讲正文模板

```markdown
# 第X章 知识串讲：{一句话概括整章的核心问题}

## 一、一句话回答
{用最短的话回答标题的问题。10-30字。}

---
## 二、{第一个叙事板块}
...

## 三、{第二个叙事板块}
...

...

## 末、概念对照表（如有易混概念）
| 概念对 | 区别 |
|--------|------|
| A vs B | ... |

## 末+1、最重要的故事 / 考试最易考点（可选）
...
```

### 1.5 写作规则

#### 叙事重组，不按页码罗列

- **不是**把笔记按页码抄一遍
- **而是**找到该章的**逻辑主线**，按因果/时间/层次重新编排
- 同一主线的概念放在同一个叙事板块里，不分散

#### 口语化语气

- 用"你"称呼读者（"你手上 5000 块想投资"）
- 用反问句引导思考（"没有金融中介会怎样？"）
- 用大白话翻译学术概念（配合 `> ` 引用块）
- 可以写"直觉上你可能觉得…但数据告诉你…"这种转折

#### 关键概念加粗 + 英文

- 首次出现的重要概念：**中文**（English）
- 不用 `[[wiki 链接]]`（串讲是线性阅读，不是超文本跳转）

#### 善用表格和文字流程图

**表格**适合：分类对比、概念区分、方案比较、事实罗列

**文字流程图**适合：因果链、推导步骤。格式：

```
原因 A
  → 中间结果 B
  → 最终结果 C
```

#### 两个标配收尾

| 收尾板块 | 何时写 | 说明 |
|----------|--------|------|
| 概念对照表（易混点） | 章内有容易混淆的概念对 | 表格：概念对 \| 区别，≥3 对 |
| 最重要的故事 / 考试最易考点 | 有考试价值的推导链 | 文字流程图复述 1-2 条核心因果链 |

#### 长度控制

- 一章 25-40 页课件 → 串讲 80-200 行 markdown
- 概念密集章可到 250 行，稀疏章 60-80 行即可

### 1.6 执行流程

```
整章笔记完成
    ↓
1. Read 该章全部笔记 + 章节索引，把握全貌
    ↓
2. 提炼逻辑主线（1-2 条核心问题）
    ↓
3. 将概念分组到叙事板块（每组 3-8 个概念）
    ↓
4. 按模板写作（口语化、叙事流）
    ↓
5. 检：关键概念是否都有英文？易混点是否覆盖？
    ↓
6. 写入章文件夹，报告完成
```

### 1.7 红线

- 🚫 **不新增知识**：串讲只重组已有笔记内容，不引入课件没讲的概念
- 🚫 **不逐页罗列**：串讲是叙事不是目录
- 🚫 **不用 wiki 链接**：串讲是线性讲稿，`[[]]` 会打断阅读流
- 🚫 **不替代章节索引**：串讲和索引是互补的——索引是地图（按页码查），串讲是导游（跟着走一遍）

- 财政学：10 篇（ch01, ch03-11）

## 二、外部 AI 讨论内容入笔记

> **触发**：人类发来与其他 AI（豆包、ChatGPT 等）的 Q&A 讨论
> **来源**：原 `笔记制作SOP.md` §第三步补充

### 2.1 基本原则

1. **课件原文是唯一权威** — 外部 AI 讨论内容必须先与课件原文交叉验证。与课件冲突的 → 不采纳；课件没涉及的 → 标记来源；与课件一致的 → 精炼后补充
2. **验证后再落笔** — 课件有没有相关的表述支持？验证通过才能写入笔记
3. **精炼，不照搬** — 对话体压缩为笔记风格（要点式、学术化、去废话），不能整段黏贴
4. **不新建笔记** — 优先找已有笔记补充
5. **不改写课件原文** — 外部 AI 内容标注来源，不与课件原文混淆
6. **找最自然的位置** — 插入笔记模板中语义最匹配的板块

### 2.2 执行流程

```
人类发来外部AI的Q&A
        ↓
1. 识别核心概念名 → Grep/Glob 搜 Vault 找到目标笔记
        ↓
2. Read 目标笔记全文
        ↓
3. 验证：对照课件原文检查外部 AI 内容
        ↓
4. 精炼：对话体 → 笔记体（压缩到原文的 1/3~1/5 篇幅）
        ↓
5. 判断归属板块 → 用 Edit 精确插入
        ↓
6. 标注来源：> 📎 以上内容来自人类与 [AI名称] 的讨论，经 Claude 验证课件原文并精炼。
```

### 2.3 常见内容类型 → 落点速查

| 外部讨论内容 | 落点板块 |
|-------------|----------|
| "为什么图中没有 XX 线" | 🧠 机制 |
| 概念 A 和 B 的区别 | ⚖️ 易混点对照 |
| 公式推导细节 | 📐 公式与计算 |
| 现实案例 | 🏛️ 应用 |
| 纯定义确认/澄清 | 📌 定义 |
| 缩写/术语的全称补全 | 各板块首次出现处 |

### 2.4 红线

- 🚫 不验证就直接写入
- 🚫 照搬外部 AI 原文（对话体）
- 🚫 不加来源标注就混入课件原文
- 🚫 用外部 AI 讨论"纠正"课件原文
- 🚫 外部提到的新概念自行扩展

## 四、图片协作（人类粘贴/替换 → AI 处理）

> **触发**：人类在 Obsidian 粘贴图片后发来 `![[Pasted image ...]]`，或说"XX的图放错了"
> **详见**：`人机协作模式手册.md` §一

### 4.1 人类粘贴 → AI 重命名归档

1. 搜 Vault 根目录找 `Pasted image *.png`
2. 搜哪个 `.md` 引用了它 → 确定上下文
3. 起描述性名称：`{课程名}Ch{XX} {描述}.png`
4. 复制到 `999-Attachments/`，更新笔记引用，补图形说明
5. 提醒人类删 Vault 根目录原图

### 4.2 人类要图 → AI 从补充源提取

1. 读笔记确定缺什么图
2. 优先从第二补充笔记（人类筛选过的）定位
3. 可编辑 PDF → `get_images()` 提取嵌入原图；矢量 → drawings 边界框裁切
4. 命名归档，插入笔记

### 4.3 人类替换 AI 的图 → AI 配合重命名

人类对源材料更熟悉，换图不辩解。配合重命名归档即可。

## 五、内容补充与修复

> **触发**：人类发来知识片段 / 说"没看懂" / 指出链接缺失
> **详见**：`人机协作模式手册.md` §二、§三

### 5.1 人类发内容 → AI 补笔记

1. 读目标笔记 → 找到最匹配板块
2. 用 Edit 精确插入
3. 引入新概念 → 搜 Vault → 已有则链，无则建壳
4. 补双向链接

### 5.2 人类说没看懂 → AI 重写

1. 读笔记定位问题节
2. 回查学习笔记原文补全信息
3. 重写该节（加定义/机制/链接）
4. 涉及图形 → 执行 4.2

### 5.3 链接缺失 → 全量修复

1. Glob 确认哪些概念已有笔记
2. 已有者补 `[[链接]]`，缺失者建壳笔记
3. 补回链（被链接笔记的 📖 相关术语）
4. Grep 验证全链

### 5.4 重复笔记 → 合并去重

1. 对比两篇：来源/importance/完整度
2. 保留更完整版本，合并独特内容（标 📎）
3. 删除精简版，更新 source 字段
