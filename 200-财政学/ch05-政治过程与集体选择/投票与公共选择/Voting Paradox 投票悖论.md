# 投票悖论 (Voting Paradox)

## 📌 定义
投票悖论（孔多塞悖论）是指在多数票规则下，即使每个投票人的偏好是理性的（可传递的），社会偏好的加总结果也可能出现非传递性——A 胜 B、B 胜 C、C 又胜 A 的循环——导致投票无法产生稳定的胜者。

> 💬 通俗理解：A比B受欢迎，B比C受欢迎——按理说A应该比C受欢迎吧？但在多数票投票中，居然可能出现C比A受欢迎的结果。投票结果取决于投票顺序，谁控制议程谁就控制结果。

## 🧠 机制

### 投票有效的条件

以三人三方案投票为例，假定偏好排序如下：

| 选择 | 亚当 | 夏娃 | 上帝 |
|------|------|------|------|
| 第一 | A | C | B |
| 第二 | B | B | C |
| 第三 | C | A | A |

无论安排怎样的投票顺序（先AB再与C竞争，或先AC再与B竞争），**B总会胜出**——此时投票是有效的。

### 投票悖论的产生

当偏好排序改变为以下结构时：

| 选择 | 亚当 | 夏娃 | 上帝 |
|------|------|------|------|
| 第一 | A | C | B |
| 第二 | B | A | C |
| 第三 | C | B | A |

- 先AB投票：A胜出 → A与C投票 → **C胜出**
- 更换次序，先AC投票：C胜出 → C与B投票 → **B胜出**

投票次序不同，结果不同——这就是投票悖论。

### 投票悖论的解除

[[Logrolling 互投赞成票|互投赞成票]]（投票交易）是解除投票悖论的途径之一。其他途径包括：
- **打分投票（基数论）**：改为打分而非排序，投票悖论自动解除
- **相对多数投票**：放弃绝对多数要求，仅取相对多数
- **代议制缩小选择集**：减少备选项目，将多个方案改为两个方案对决

## 🏛️ 应用

投票悖论的现实含义：在多数票规则下，**"议程操纵"是巨大的权力**——谁设定选举程序或议事程序，谁就可以影响甚至决定结果。这使得议事规则的设计成为政治制度的核心议题。
