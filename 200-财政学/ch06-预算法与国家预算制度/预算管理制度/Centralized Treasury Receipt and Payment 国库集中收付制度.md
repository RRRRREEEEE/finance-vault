# 国库集中收付制度 (Centralized Treasury Receipt and Payment)

## 📌 先分清三个概念（最易混淆）

| 概念 | 范围 | 一句话 |
|------|------|--------|
| **国库单一账户制度** | 国外标准叫法 | 所有财政资金只存在国库单一账户，收支全流程统一管控 |
| **国库集中收付制度** | = 集中收缴 + 集中支付 | 我国完整制度，收入支出两端全覆盖，对标国库单一账户制度 |
| **国库集中支付制度** | 仅支出半边 | 只改花钱流程，不管收入收缴，是改革第一阶段的过渡方案 |

> ⚠️ 原文将三者混用。区分关键：**集中收付 = 收+付；集中支付 = 只有付**。

## 🧠 改革两步走（为什么先支付、后完整收付？）

### 旧制度弊病（改革前）
```
财政拨款 → 单位对公账户存放 → 单位分批付款
```
资金长期沉淀在各单位账户、挤占挪用、监管盲区大。

### 第一步：国库集中支付（只改支出端，优先解决最大痛点）

**改革动作：**
1. 取消单位实存资金账户，改用**零余额账户**（账户额度≠现金，当天清零）
2. 两种支付方式：
   - **财政直接支付**：工资、大额采购 → 财政局直接打给收款人
   - **财政授权支付**：小额零星开支 → 单位在限额内申请支付
3. 资金**不过单位手**，当天清算回国库，全程监控去向

**局限性：** 只管住了"花钱"，**收入收缴还是老模式**——税务局、收费单位仍有过渡账户，收费先存单位户、隔一段时间才上缴国库，依旧存在坐支、截留非税收入的漏洞。只有支出管控，制度不完整。

### 第二步：国库集中收付（收入+支出双改革，最终目标）

在集中支付基础上叠加**国库集中收缴**，补齐收入端：

**收入端（集中收缴）：**
- 税收：财税库银联网，纳税人直接缴税进国库
- 非税收入（办证费、罚款、土地款等）：执收单位**只开票据、不收钱**，缴款人直接缴入国库/财政专户
- 彻底取消所有征收部门过渡账户，从源头杜绝截留、坐支

**支出端（集中支付）：** 沿用第一步已建好的体系

**结果：** 全部财政收支（四本预算资金）统一纳入国库单一账户体系，收支两头全流程实时监控——这才等同于西方的国库单一账户制度。

## 📊 两步走对比

| | 国库集中支付（第一步） | 国库集中收付（第二步/最终目标） |
|------|------|------|
| 覆盖范围 | 仅**支出**环节 | **收入+支出**全流程 |
| 收入管理 | 沿用旧模式，存在单位过渡账户 | 集中收缴，无过渡户，资金直缴国库 |
| 支出管理 | 零余额账户、直接/授权支付 | 同左，支出体系成熟 |
| 完整度 | 不完整、阶段性改革 | 完整闭环，等同于国库单一账户制度 |
| 核心解决问题 | 管住花钱，防止挤占挪用 | 收支两头全管控，收入截留+支出乱象一并根治 |

## 🔗 三大预算执行改革区分

国库集中收付、政府采购、收支两条线并称三大预算执行改革，各管一段：

| 改革 | 管什么 | 怎么管 |
|------|--------|--------|
| **政府采购制度** | 规范**采购行为** | 统一招标，公开竞争 |
| **收支两条线** | 切断**收入利益链** | 收缴分离、收支脱钩，是集中收缴的前置配套 |
| **国库集中收付** | 管**资金存放与缴拨路径** | 资金全程不入单位自有账户，国库直达 |

## 📄 来源原文
> 我国国库制度改革的最终目标是借鉴西方国家实行的"国库单一账户制度"，即实行集中收付制度。第一步先实行集中支付制度，而后再实现国库集中收付制度。国库集中支付制度，就是对预算资金分配、资金使用、银行清算及资金到达商品和劳务供应者账户的全过程集中进行全面监控的制度。

## 📝 简答背诵

1. **国库集中收付制度（国库单一账户制度）** = 集中收缴 + 集中支付，对财政收支全流程监控
2. **改革分步**：第一步推行国库集中支付（仅规范支出）；第二步完善收入集中收缴，建成完整国库集中收付制度，全部财政资金纳入国库单一账户统一管理
